# features仕様

## 責務

`src/features/`は、生成HTMLへ追加する機能単位のマークアップ、スタイル、ブラウザJavaScriptを所有する。ブラウザスクリプトは外部通信やCDNに依存せず、生成HTML内へ埋め込める文字列として返す。

## ファイル

### `code-blocks/index.ts`

- `createCodeBlocksFeature(renderer, highlight)`: markedのcode rendererを設定し、コードブロックの有無を追跡する。
- `renderCodeBlock()`: 言語ラベル、`折り返す`、`コピー`操作を含むコード領域を生成する。
- `renderCodeBlockScript()`: コピーと折り返し切り替えをイベント委譲で実装する。
- `CodeBlocksFeature.renderScript()`: コードブロックが存在する場合だけスクリプトを返す。

対応言語はhighlight.jsで強調し、未対応言語と強調無効時はHTMLエスケープしたプレーンコードを出力する。コピー操作には共通のinline SVGを表示し、Clipboard APIが使えない場合または書き込みを拒否した場合にtextareaと`execCommand("copy")`へフォールバックする。

### `file-tree/index.ts`

- `createFolderIds()`: ルート相対フォルダパスのSHA-256をbase64url化し、6文字から始めてツリー内で一意になるまで全IDを延長する。
- `renderNodes()`: ディレクトリを短縮フォルダIDとフォルダSVG付きの`details`、ファイルをリンクとして再帰描画する。ファイルには有効コメント件数、ディレクトリには配下の合計件数を1件以上の場合だけバッジ表示する。
- `renderFileTree(options)`: パンくず直下のポップオーバーとして、ファイルツリーとファイル名フィルターを生成する。
- `renderFileSidebar(options)`: 画面左端へ常設するヘッダー、GitHub風のペイン開閉アイコン、ファイル名フィルター付きファイルツリーを生成する。
- `renderFileTreeScript(enabled)`: 常設サイドバーとポップオーバーの開閉、フォルダ状態の相互同期、URL保存、個別フィルター、Escapeによる解除、祖先ディレクトリ表示、現在ファイルパスのコピーを実装する。
- `renderBreadcrumbs()`: リンク可能セグメントと現在文書からパンくずを生成し、閉じたファイルサイドバーを開くボタン、現在ファイル名とchevronだけを持つリンク風のポップオーバー開閉トリガー、枠なしのパスコピーボタン、右端の指定更新日を置く。現在ファイルの直前には祖先セグメントがある場合だけ`/`区切りを置き、ルート文書では区切りと空のパンくずリストを省略する。パンくずがなくてもファイルサイドバー開閉ボタンと`ファイル`表示のポップオーバートリガーを生成する。
- `renderModifiedAt()`: ISO 8601更新日時を検証し、UTCの日時を`更新 YYYY-MM-DD HH:mm:ss`形式の`time`要素として生成する。未指定時は何も生成しない。
- `renderModifiedAtScript()`: `time`要素のISO日時をブラウザのローカルタイムゾーンへ変換し、`更新 YYYY-MM-DD HH:mm:ss`形式で表示する。タイムゾーン名とUTCオフセットは表示しない。JavaScript実行前や無効時はUTCの日時を同じ形式で維持する。

ファイルサイドバーはデスクトップで画面左端へ全高の固定ペインとして初期表示し、角丸や外周のカード枠を付けず右境界線だけで本文と分ける。開閉トグルは開閉どちらの状態でも固定ヘッダー左端へ同じ位置に前面表示し、閉じた場合は本文を左へ拡張する。ヘッダーのブランド表示はトグルと重ならない左余白を持つ一方、サイドバー見出しにはトグル用の空白を設けない。パンくず内では現在ファイル名を`sample.md ▼`のようなリンク風トリガーとして表示し、従来のボタン装飾やフォルダアイコンは使わずにポップオーバーを開く。隣接するパスコピーアイコンは正方形ボタンの中央へ配置する。サイドバーの閉状態は`file-sidebar=closed`クエリパラメータへ保存する。900px以下では初期表示を閉じ、開いた場合は本文幅を変えない固定オーバーレイとする。ポップオーバーは外側クリックまたはEscapeで閉じ、開閉状態を`marksites-files=open`へ保存する。両状態は同一ホスト・プロトコルのHTMLリンクへ引き継ぐ。

常設側とポップオーバー側は同じ短縮フォルダIDを持ち、片方でのフォルダ開閉をもう片方へ同期する。フォルダの開状態は短縮IDごとの反復可能な`open`クエリパラメータへ保存する。同じフォルダパスはページ間・再ビルド後も同じIDとなり、衝突時はID長を延長する。パラメータ未指定時は全フォルダを閉じ、検索による一時的な開閉は保存しない。各表示の検索欄は独立して動作し、大文字小文字を区別せず`data-file-name`のファイル名へ部分一致する。コメント件数は検索文字列に含めず、一致ファイルの祖先だけを表示し、検索解除時は保存済みの開閉状態へ戻す。コメント0件のバッジは表示しない。パスコピーはセキュアコンテキストでClipboard APIを優先し、利用不可または書き込み拒否時は一時textareaと`execCommand("copy")`へフォールバックする。名前、href、ラベル、コピー対象属性はHTMLエスケープする。

常設ファイルサイドバーの幅は280pxとする。開閉トグルは枠線と通常背景を持たず、hover時の背景とキーボードフォーカスリングだけを表示する。

### `table-of-contents/index.ts`

- `createTableOfContentsFeature(renderer, config)`: markedのheading rendererを設定し、GithubSluggerで決定的な見出しIDを生成する。
- `renderTableOfContents()`: 指定深度内の見出しを階層インデント付きリンクへ変換する。
- `renderTableOfContentsScript()`: レスポンシブ開閉とスクロール位置に応じた現在見出しの強調を実装する。

同一見出しにはGitHub互換の接尾辞を付ける。見出しリンクと現在位置の追従を担当し、サイドバー全体のタブ・レスポンシブ開閉は`sidebar.ts`へ委ねる。

### `sidebar/index.ts`

- `createSidebarFeature()`: デフォルトで`目次`と表示する目次と現在ページのコメント一覧を右サイドバーのタブとして構成し、マークアップ、スタイル、ブラウザ動作を返す。
- `activate()`: ARIA tabの選択状態と対応するtabpanelの表示を同期する。
- `setExpanded()`: 900px以下でサイドバー全体を折り畳む。

デスクトップではタブヘッダーを固定し、選択中のパネルだけを独立スクロールさせる。stickyのtopは固定ヘッダー56pxと本文上余白32pxを加えた88px、モバイルでは68pxとし、スクロール開始時の位置補正を発生させない。左右矢印キーでタブを移動でき、コメント作成・本文ハイライト選択時の`marksites:show-comments`イベントでコメントタブを開く。

### `header/index.ts`

- `createHeaderFeature()`: 画面上端へ固定する`marksites`ブランド、右端の枠なしテーマ切り替え・言語切り替えアイコン、スタイル、ブラウザ動作を返す。文書名はパンくずと本文へ委ね、固定ヘッダーには重複表示しない。

テーマ切り替えはOS設定を初期値とし、明示的な選択を`theme=dark|light` GETパラメータへ保存する。`data-theme`と配色変数をbodyへ設定し、ライト・ダーク双方で本文、左右サイドバー、操作ボタンの色を同期する。言語切り替えは日本語を既定とし、英語選択時だけ`lang=en`を保存する。文書本文、見出し、ファイル名、コメント本文などのユーザーコンテンツは変更せず、生成UIのラベル、ツールチップ、ARIAラベル、プレースホルダーと動的に追加されたコメント操作だけを翻訳する。両パラメータは同一ホスト・プロトコルのHTMLリンクへ引き継ぐ。

ヘッダー高は56pxとし、本文と右サイドバーの上位置をヘッダー下32pxの88pxで揃える。ファイルサイドバーの有無と開閉状態にかかわらず同じ上余白を維持し、見出しのスクロール余白、左ファイルペインの開始位置も補正する。900px以下では文書名を省略して操作領域を確保する。

### `annotations/index.ts`

- `safeJson()`: `<`、`>`、`&`をUnicode escapeし、JSON script要素からの脱出を防ぐ。
- `createAnnotationsFeature(data?)`: 現在文書のコメントデータ、`コピー`、`AI向けコピー`、＋アイコン付き`コメント`を横並びにした選択操作ツールバー、サイドバーへ渡すコメントパネル、件数、スタイル、ブラウザ動作を一体で返す。

ブラウザ側の主要処理:

- `selectionData()`: 本文内の選択だけを受け付け、exact、prefix、suffix、headingId、offsetを作る。
- `copy()`: 選択文字列、AI用Markdown、コメント情報をコピーし、Clipboard APIの利用不可または書き込み拒否時のフォールバックを持つ。
- `annotationMetadata()` / `allCommentsMetadata()`: 文書パス、直前の章見出し名、引用、コメント本文だけを含む可読テキストを、単一コメントまたは新しい順の全コメントについて組み立てる。HTML依存の見出しIDと位置、コメント属性の状態、画面表示用の作成・更新日時はコピーへ含めない。文書全体コメントは空の引用位置を検索せず、見出しを`なし`、引用を`文書全体`とする。古い引用コメントに見出しIDがない場合は、再特定した引用位置から直前の見出しを補完する。
- `locate()`: 保存済み見出し内を優先し、完全一致、前後文、保存オフセットの順でコメント位置を再特定する。見出し名の変更や引用の移動に備え、見出し内で見つからなければ文書全体を検索する。
- `classify()`: 保存状態から`comments`または`archived`の表示グループを決め、有効コメントについて引用、文書全体、引用先不明の表示種別を判定する。引用先不明は永続化せず、Markdownの変化に応じて再判定する。
- `render()`: 常時展開する有効コメントと折り畳み可能なアーカイブ、有効件数、本文ハイライトを描画し、位置不明は誤配置せず表示する。
- `updateCurrentFileCount()`: コメント作成・アーカイブ・復元・削除後、再読み込みを待たず常設側とポップオーバー側にある現在ファイルと全祖先フォルダのツリーバッジを同期する。
- `showAnnotation()`: 一覧から本文の対象箇所へ移動し、本文ハイライトからは対応カードを表示する。
- `connect()`: HTTP時だけruntime APIを確認し、能力が一致した場合だけ編集を有効化する。
- `mutate()`: 作成・編集・アーカイブ・復元・削除をAPIへ送り、409時は最新状態を再取得する。

一覧の先頭に＋アイコン付き`追加`とコピーアイコン付き`コピー`を左右へ並べ、コピー操作を各カードの操作列の先頭にも置く。上部コピーはツールチップとARIAラベルで有効コメントだけが対象であることを示す。操作結果のステータス行は空の間は非表示にして、操作領域とコメント一覧の間に不要な余白を作らない。アーカイブ済みコメントは一括コピーとコメント・ファイル・フォルダのバッジ件数から除外する。コピーは編集可否にかかわらず利用できる。一括アーカイブ操作は設けない。

有効コメントは見出しや囲みを追加せず常に展開し、`アーカイブ`だけを見出し付きの折り畳みとして初期状態で閉じる。アーカイブ見出しには件数を表示しない。コメントパネルは表示中の内容に応じた高さとし、サイドバーの画面内最大高を超える場合だけスクロールさせる。アーカイブを閉じた場合は非表示になった内容の高さを残さない。アーカイブを開いた状態は`comments=archived` GETパラメータで保存し、HTMLページ間のリンクへ引き継ぐ。アーカイブ済みカードはコメントをコピー、コメントを復元、コメントを削除、それ以外はコメントをコピー、コメントを編集、コメントをアーカイブ、コメントを削除の操作を持つ。復元後は有効コメント一覧へ戻り、引用、文書全体、引用先なしの種別は現在のMarkdownから再判定する。削除は確認後にJSONから物理削除する。

新規追加時のフォームは追加ボタン直下かつコメント一覧の直前へ移動し、編集時だけ対象カード内へ移動する。文書全体コメントはblockquoteを作らず`文書全体`と表示する。引用先が見つからないコメントもblockquoteにせず、`引用先なし`と保存された元の選択文字列を通常のコンテキスト情報として表示する。一覧には埋め込まれた現在ページのコメントを各グループ内で`createdAt`降順に表示し、更新日時はブラウザのローカルタイムゾーンへ変換するがタイムゾーン名は表示しない。メタ情報行は左に`引用`、`引用先なし`、`文書全体`のいずれかのバッジ、右に更新日時を表示する。新規コメントの保存後は常時展開された一覧の追加カードへキーボードフォーカスと選択状態を移して表示領域へスクロールする。既存コメントの編集では並び順を変えない。サイドバー内ではコメント本文を13px、引用文とコンテキストを12px、更新日時と表示種別を11px相当のコンパクトな文字サイズで表示し、長い文字列はDOMに保ったままCSSで2行に制限する。コメントグループとカードは角丸の枠で囲まず、サイドバーの横幅いっぱいにフラットな区切り線で表示し、末尾にサイドバー内側の余白を残さない。コメント欄内の操作ボタンは6pxの角丸と共通配色を使用する。カード操作は右上へ重ねる絶対配置とし、非表示時にカード内の領域を確保しない。カードのhover中、操作領域のキーボードフォーカス中だけ表示し、カードの選択状態では表示を維持しない。端末のhover media queryは使用せず、実際のtouch pointerでカードを選択した場合だけ操作を表示する。フォーカス中のカードと対応する本文ハイライトはアクセント背景・境界色へ変える。`file://`では変更操作を無効にしてAPIへ接続せず閲覧専用とするが、コピーはClipboard APIの代替処理で使用できる。選択操作は目次、ファイルツリー、コード操作、コメントUI自身を対象外にする。コメント本文はDOMの`textContent`で表示する。

編集時は共通フォームを対象カード内へ移動し、引用を対象箇所として残したまま本文・操作ボタンをフォームへ置き換えてその場で編集する。編集中は選択カードのアクセント背景と太いoutlineを解除し、通常背景、境界線、控えめなshadowでフォーム状態を示す。textareaは6pxの角丸とfocus ringを持ち、操作行は右寄せして`保存`をprimary、`キャンセル`をsecondaryとして表示する。保存またはキャンセル後は通常のカード表示へ戻す。カード操作は文字ラベルを持たない28pxのアイコンツールバーへまとめ、用途は日本語の`aria-label`、`title`、フォーカス時とhover時に表示するツールチップで示す。ツールチップがカード領域で切れないよう、コメントグループはoverflowを可視にする。削除操作はhover時だけdanger色で区別する。

### `image-viewer/index.ts`

Markdown本文内の画像をクリックまたはEnter/Spaceで全画面表示する。ツールバーで拡大、縮小、100%へのリセット、終了を操作でき、ホイールによるズームとpointer操作によるドラッグ移動にも対応する。終了操作はEscapeと閉じるボタンだけとし、開始元の画像へフォーカスを戻す。CSS、マークアップ、JavaScriptはHTMLへ埋め込み、外部依存を持たない。

## 依存境界

- `types.ts`はFeature間で共通する内部契約だけを定義し、公開型は置かない。
- 複数のマークアップとブラウザ動作を持つFeatureは専用フォルダを所有し、`index.ts`を内部エントリポイントにする。
- Feature固有の追加ファイルはそのFeatureフォルダ内に置き、`utils/`へ移すのは複数Featureから共有する副作用のない処理だけとする。

- featureはファイルシステムへアクセスしない。
- 永続化、HTTPルーティング、ディレクトリ探索は扱わない。
- feature固有のCSSとスクリプトは可能な限り同じファイルで管理する。
