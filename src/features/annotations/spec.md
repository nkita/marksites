# annotations feature仕様

## 責務

生成HTMLに埋め込む注釈UI、そのスタイル、ブラウザ動作を所有する。Node側の注釈モデルと永続化は`src/annotations/`が所有する。

## ファイル

- `index.ts`: 注釈Featureを構成する内部エントリポイント。

ブラウザUI固有の型を公開APIへ漏らさない。

コメント本文、引用、編集フォーム、本文ハイライトはテーマ変数を使用する。ライトテーマ専用の文字色や黄色背景を固定せず、ダークテーマでも本文との区別と文字の可読性を維持する。

文書全体コメントはカード上部へ重複ラベルを置かず、メタ情報の`文書全体`バッジだけで示す。コピー用テキストの生成時も空の引用文字列を本文から検索せず、見出しを`なし`、引用を`文書全体`として直接組み立てる。引用先を特定できない場合は保存された引用元を取り消し線付きで表示し、`引用先なし`バッジと組み合わせる。

上部操作は`追加`と`コピー`を左右へ並べる。コピーは有効なコメントだけを対象とし、ツールチップとARIAラベルでも対象を明示する。アーカイブはURLに開状態がない初期表示では折り畳み、帯全体を操作・hover領域にする。アーカイブが0件なら帯を表示せず、全コメントが0件なら単一の空状態だけを表示する。

選択範囲、AI向けテキスト、単一コメント、一括コメントのコピーは同じ処理を使用する。セキュアコンテキストでClipboard APIを優先するが、APIの存在だけを成功条件とせず、権限拒否を含む書き込み失敗時も一時textareaと`execCommand("copy")`へフォールバックする。両方が失敗した場合だけ失敗として表示する。
