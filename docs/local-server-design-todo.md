## 文書の目的

この文書は、[ローカルサーバー・コメント機能 設計素案](./local-server-design.md) の第1段階から第4段階までを、実装可能なタスクへ分解したチェックリストである。

タスクは原則として上から実施する。各段階の完了条件を満たすまでは次の段階を開始しない。将来の検討候補は、このTODOの対象に含めない。

## 共通ルール

- 開発に関する設計、仕様、検討結果、実装判断はMarkdownとして `docs/` 配下に格納する。
- 新しい機能領域の設計文書を追加する場合は、内容が分かるkebab-caseのファイル名を使用する。
- 実装中に設計変更や新しい判断が発生した場合は、コード変更と同じ作業内で対応する `docs/` 配下の文書を更新する。
- 確定した仕様は設計文書へ、未実施の作業と完了状況はTODO文書へ記録し、両者を混在させない。
- READMEには利用者向けの使用方法を記載し、詳細な内部設計は `docs/` 配下の文書へリンクする。
- `markdownToHtml()` と `src/index.ts` から公開済みの型を壊さない。
- 既存CLIの `marksites <input> [output]` を維持する。
- ESM importには `.js` 拡張子を付ける。
- 生成HTMLは引き続き単体で成立し、`file://` から閲覧できるようにする。
- コメントJSONを唯一の正本とし、IndexedDBは使用しない。
- コメントJSONとビルド管理ファイルは、既存データを壊さない方法で書き込む。
- 新しい内部モジュールはpackageの `exports` から公開しない。
- 各タスクで必要なテストを追加し、段階完了時に `npm test`、`npm run check`、`git diff --check` を実行する。
